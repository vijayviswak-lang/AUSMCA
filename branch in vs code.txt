 By default, your repository starts with a main or master branch, but as your project grows, you'll often need to work on new features, 
bug fixes, or experiments without disrupting the main code. 
This is where branching comes in, allowing you to create isolated workspaces.

Why Use Git Branches?
Using branches in Git provides multiple benefits:

Parallel Development: Teams can work on various parts of a project at the same time without causing conflicts.
Safe Experimentation: You can test new features or fixes in an isolated environment without impacting the main codebase.
Version Control: Branches allow you to keep your production environment stable while still developing and testing new ideas.
